# Registry Workflow

This project uses a shadcn-compatible registry as the source of truth for docs and distribution.

## Source of truth

- Registry items: `registry/items/*.json`
- Installable files: `registry/base-vega/<slug>/*`
- Wrapper files for DX: `components/*` (thin re-exports)

## Commands

- Add a new component: `pnpm components:add`
- Regenerate docs + registry outputs: `pnpm components:update`
- Validate metadata and dependencies: `pnpm components:check`
- Build shadcn registry payload: `pnpm registry:build`

Environment variables (optional overrides):

- `REGISTRY_NAME` (default: `base-vega`)
- `REGISTRY_ALIAS` (default: `@vega`)
- `REGISTRY_HOMEPAGE` (default: `http://localhost:3000`)
- `REGISTRY_URL_TEMPLATE` (default: `${REGISTRY_HOMEPAGE}/r/{name}.json`)

The docs pages can also use the following public env vars:

- `NEXT_PUBLIC_REGISTRY_ALIAS`
- `NEXT_PUBLIC_REGISTRY_HOMEPAGE`
- `NEXT_PUBLIC_REGISTRY_URL_TEMPLATE`

Non-interactive usage for agents:

- `pnpm components:add -- --slug my-component --title "My Component" --description "..." --type registry:component --sourcePath registry/base-vega/my-component/my-component.tsx --usagePath registry/base-vega/my-component/usage.tsx --previewPath registry/base-vega/my-component/preview.tsx --wrapperPath components/ui/my-component.tsx`

## Generated files

Do not edit these manually:

- `lib/component-registry.generated.ts`
- `components/docs/component-previews.generated.tsx`
- `registry.json`

## Conventions

- Every file listed in `registry/items/*.json` must live under `registry/base-vega/<slug>/`
- Registry source files should import from `@/registry/base-vega/...` for local registry dependencies
- Docs metadata belongs under `meta.docs` in each item file
- Wrapper files are optional and are generated by `components:add` unless `--no-wrapper` is used

Install targets:

- `components/ui/*` wrappers default to `registry:ui`
- `components/backgrounds/*` and `components/reactive-svg/*` wrappers use explicit `target`
